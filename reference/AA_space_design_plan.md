# AA 스페이스 설계 및 운영 정책 계획서

> 작성일: 2026-06-15 | 대상 스페이스: AA (덴탈AI연구소 운영 아카이브)  
> 목적: MPS 계획·평가 자동화를 위한 Confluence 신규 스페이스 설계

---

## 배경 및 목적

### 왜 신규 스페이스가 필요한가

현재 SD 스페이스(Digital R&D Center)는 661개 페이지와 75개 폴더가 혼용된 구조로,
LLM/RAG 기반 MPS 자동화 워크플로우의 데이터 원천으로 사용하기에는 다음 문제가 있습니다:

- 레이블 사용률 **0%** → 카테고리 분류·검색 불가
- 일상 기록(Daily Scrum 등)과 전략 문서가 동일 계층에 혼재 → RAG 노이즈
- MPS 문서가 4개 팀폴더에 분산 → 비교·집계 불가
- 올드 페이지·빈 Archived 컨테이너 → 검색 품질 저하

**AA 스페이스**는 "MPS 계획·평가에 실제로 활용 가능한 정보만" 선별·정제하여 보관하는
**큐레이션된 지식 허브**로 운영합니다.

## 1. AA 스페이스 폴더 구조 설계

### 설계 원칙

| 원칙 | 내용 |
|------|------|
| **최대 깊이 3단계** | 루트 폴더 → 카테고리 → 문서 |
| **페이지 전용** | 폴더(Confluence folder) 사용 최소화, 페이지 계층으로 구성 |
| **목적 중심 분류** | 문서 종류(MPS/프로젝트/조사 등)가 최우선 기준 |
| **연도 보조 분류** | 연도는 하위 계층에서만 사용 (루트 기준 아님) |
| **레이블 필수** | 모든 페이지에 최소 3개 레이블 부착 (팀, 문서유형, 연도) |

### 2-2. 이관 제외 (가져오지 않을 페이지)

아래 조건 중 **하나라도 해당**하면 이관하지 않습니다.

| 제외 조건 | 대표 예시 |
|-----------|-----------|
| ❌ Daily Scrum 관련 페이지 전체 | `Daily Scrum > 2024-Daily Scrum` 하위 전체 |
| ❌ 2024년 이전 생성 + 2024년 이전 최종 수정 | `Old > R&D MPS`, `Old > Solution MPS` 등 |
| ❌ 빈 Archived 컨테이너 (12개) | `Archived Robot vLEg`, `Archived AI KdeG` 등 |
| ❌ 결론 없는 단순 회의록 | 액션아이템 없이 대화만 기록된 페이지 |
| ❌ 내부 링크·임시 메모 성격 | 내용이 1~3문장 이하의 링크 모음 페이지 |
| ❌ 완료되지 않은 실험 기록 | 결과 없는 테스트 노트, WIP Draft |
| ❌ 중복 페이지 (구버전) | Old 폴더에 중복 존재하는 구버전 MPS |

> [!NOTE]
> **"evergreen" 예외**: 작성일이 오래되었더라도 팀 방향성·개발 철학을 담은 문서는  
> `status:evergreen` 레이블을 붙여 예외적으로 이관합니다.  
> (예: How To Develop 폴더의 Git 전략 가이드, 개발 방법론 등)

---

### 2-3. 이관 방식

- **복사(Copy)**: SD 스페이스 원본은 그대로 유지, AA 스페이스에 복사본 생성
- **레이블 부착**: 복사 후 즉시 레이블 부착 (아래 레이블 체계 참고)
- **원본 링크 표기**: 각 AA 페이지 하단에 `SD 원본: [링크]` 메타데이터 추가
- **이관 일자 기록**: 페이지 속성 또는 헤더에 `이관일: YYYY-MM-DD` 명시

## 3. 레이블 체계 (필수)

모든 AA 스페이스 페이지는 아래 4가지 레이블 그룹에서 **최소 1개씩, 총 3개 이상** 부착합니다.

### 3-1. 팀(Team) 레이블

| 레이블 | 대상 |
|--------|------|
| `team:ai` | AI 팀 관련 문서 |
| `team:sw` | SW 팀 관련 문서 |
| `team:device` | Device 팀 관련 문서 |
| `team:solution` | Solution 팀 관련 문서 |
| `team:rnd` | R&D 팀 관련 문서 |
| `team:center` | 팀 공통 / 센터 전체 |

### 3-2. 문서유형(DocType) 레이블

| 레이블 | 대상 |
|--------|------|
| `doctype:mps-annual` | 연간 MPS |
| `doctype:mps-monthly` | 월간 MPS |
| `doctype:mps-weekly` | 주간 MPS/보고 |
| `doctype:project-status` | 프로젝트 현황 |
| `doctype:tech-survey` | 기술 조사·분석 |
| `doctype:market-survey` | 시장·경쟁 조사 |
| `doctype:guideline` | 가이드·프로세스 문서 |
| `doctype:patent` | 특허·논문 |
| `doctype:gov-project` | 정부과제 관련 |

### 3-3. 연도(Year) 레이블

| 레이블 | 대상 |
|--------|------|
| `year:2024` | 2024년 생성/유효 문서 |
| `year:2025` | 2025년 생성/유효 문서 |
| `year:2026` | 2026년 생성/유효 문서 |

### 3-4. 상태(Status) 레이블

| 레이블 | 대상 |
|--------|------|
| `status:active` | 현재 진행 중·유효한 문서 |
| `status:completed` | 완료된 과제·MPS |
| `status:evergreen` | 연도 무관 상시 참조 문서 |
| `status:review-needed` | 유효성 재검토 필요 (6개월+ 미수정) |

## 4. 명명 규칙 표준화

### 4-1. MPS 문서

```
[팀] YYYY-MM 월간 MPS
[팀] YYYY 연간 MPS
[전체] YYYY-WNN 주간 보고  (NN = 주차)
```

예시:
- `[AI팀] 2026-01 월간 MPS`
- `[SW팀] 2025 연간 MPS`
- `[전체] 2026-W23 주간 보고`

### 4-2. 프로젝트 현황 문서

```
[과제명] 현황 요약 (YYYY)
[과제명] 로드맵 (YYYY)
```

예시:
- `강원지역혁신클러스터 현황 요약 (2026)`
- `AI 과제 관리 시스템 로드맵 (2026)`

### 4-3. 기술 조사 문서

```
[주제] 기술 조사 (YYYY-MM)
[주제] 비교 분석 (YYYY-MM)
```

예시:
- `RAG 프레임워크 기술 조사 (2025-09)`
- `IDS 2025 전시회 분석`

### 4-4. 가이드라인

```
[주제] 가이드라인 vX.X
```

예시:
- `Git 형상관리 가이드라인 v2.0`
- `MPS 작성 프로세스 가이드라인 v1.0`

## 5. 유지보수 정책

### 5-1. 상시 정책 (신규 페이지 생성 시)

| 규칙 | 내용 |
|------|------|
| **위치 규칙** | 문서 유형에 맞는 폴더에 생성. 위치 위반 시 Dify 에이전트가 Slack으로 이동 요청 |
| **레이블 규칙** | 생성 후 24시간 이내 레이블 미부착 시 Dify 에이전트가 자동 분석 후 태그 제안 |
| **명명 규칙** | 표준 명명 패턴 미준수 시 알림 (강제 변경 없음) |

### 5-2. 정기 정책 (주기적 검토)

| 주기 | 작업 | 담당 |
|------|------|------|
| **월간** | 신규 이관 페이지 레이블 검증 | Dify 에이전트 |
| **분기** | `status:active` 페이지 중 3개월+ 미수정 → `status:review-needed` 변경 | Dify 에이전트 |
| **반기** | 6개월 이상 미수정 페이지 작성자에게 "유효성 확인" 알림 | Dify → Slack |
| **연간** | 전년도 MPS 페이지 `status:completed` 처리, 구조 정비 | 스페이스 관리자 |

### 5-3. 문서 유형별 유효기간 정책

| 문서유형 | 유효기간 | 만료 후 처리 |
|---------|---------|-------------|
| 월간 MPS | 무기한 | `status:completed` 레이블, 보관 |
| 연간 MPS | 무기한 | `status:completed` 레이블, 보관 |
| 주간 보고 | 1년 | 1년 경과 시 SD 스페이스로 이동 |
| 프로젝트 현황 | 과제 종료+6개월 | `status:completed` 처리 |
| 기술 조사 | 2년 | `status:review-needed`, 갱신 또는 삭제 |
| 가이드라인 | evergreen | 버전업 후 구버전 archived 처리 |
| 정부과제 | 과제 종료+1년 | SD 원본만 유지 |

### 5-4. Dify 자동화 에이전트 기능 (구현 목표)

| 기능 | 트리거 | 액션 |
|------|--------|------|
| **위치 검증** | 신규 페이지 생성 webhook | 폴더 위치가 정책 위반이면 Slack으로 이동 요청 |
| **자동 태깅** | 레이블 없는 페이지 감지 | 페이지 내용 분석 → 레이블 3개 추천 후 작성자 알림 |
| **만료 알림** | 정기 스캔 (매주 월요일) | 6개월 이상 미수정 페이지 작성자에게 "유효성 확인" 요청 |
| **MPS 보조** | Dify MPS 워크플로우 호출 | AA 스페이스에서 레이블 기반 관련 문서 검색 후 컨텍스트 제공 |
